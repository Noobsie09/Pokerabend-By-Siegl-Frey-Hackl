# Kurz-Dokumentation Scrum-Prozess (GitHub)

## 1. Product Backlog
Die Anforderungen wurden als **User Stories in GitHub Issues** angelegt:

- **#1 Pokerabend organisieren (Gastgeber)**
- **#2 Spielregeln festlegen (Mitspieler)**
- **#3 Snacks & Getränke organisieren (Teilnehmer)**

Jedes Issue enthält eine User Story sowie Akzeptanzkriterien.  
*Screenshot: jeweiliges GitHub Issue*

---

## 2. Sprint Planning
Alle drei Issues wurden für einen Sprint eingeplant und jeweils einem Verantwortlichen (Assignee) zugewiesen.  

---

## 3. Durchführung des Sprints
Die Bearbeitung der Issues erfolgte direkt in GitHub durch Kommentare und Statusverfolgung während des Sprints.  

---

## 4. Sprint Review
Am Ende des Sprints wurde geprüft, ob die Akzeptanzkriterien erfüllt sind. Die User Stories konnten abgeschlossen werden.  

---

## 5. Sprint Retrospektive
**Positiv:** klare User Stories, einfache Nachvollziehbarkeit  
**Verbesserung:** Nutzung eines Projekt-Boards
